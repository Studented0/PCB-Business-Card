# 5/4/2026 3 AM - PCB Layout and Routing

_Time spent: 3.25h_

Layout started with setting the board size to 100×75mm. That fits JLCPCB's economic assembly limit and is pretty similar to a SATA SSD.
The first thing I placed was the display connector. The 2.0" ST7789V module from Amazon has a footprint that takes up almost the entire top half of the board. I didn't expect that. It actually worked out well because now the card has a clear right half that's just the screen and a left half where all the components live. 
I added a copper QR code using KiCad's image converter tool. That process was a disaster. The first import came in as a giant pink rectangle covering half the board. Try again with different scale. Then the inversion was wrong, the QR was solid copper with dark cutouts instead of copper squares on green. Spent way too long trying to flip it. Eventually accepted the current version which actually looks fine as exposed copper on black PCB. Added a soldermask opening over it so it shows as bare metal.
The NFC antenna coil went on B.Cu as a two-turn rectangular loop around the board perimeter. I started routing it on F.Cu but moved it to the back so the decorative copper circles on the front wouldn't interfere with the antenna field. The circles are purely aesthetic, no net assigned, just exposed copper art. I was worried they'd kill the NFC range but at tap distance it should be fine.
Routing was honestly pretty bad. I used push-and-shove mode and just connected everything. Power traces are 0.5mm, signals are 0.2mm, and pours on both layers handle GND. There are a few ugly detours, but zero unconnected nets. 
I added four mounting holes at the corners and a lanyard slot cutout on Edge.Cuts at the top center. The slot means you can clip a lanyard directly to the board without a separate hole. Added silkscreen text for my name, Mechatronics, phone number, and GitHub handle between the display and the QR code.

# 5/4/2026 3 AM - Planning and Schematic

_Time spent: 3.5h_

I wanted to build a PCB business card but make it cool. It shows my Spotify activity from Discord using Lanyard(other Discord activities too), let people tap it with their phone to open the song I'm listening to, and have a QR code on the board that links to my GitHub(for now the its set to a bit.ly). I also wanted it to be wearable as a lanyard.
The first thing I had to figure out was the MCU. I almost went with the ESP32-C3 Mini but that would have been a mistake. The C3 doesn't have enough RAM to hold a JPEG decoder, a WiFi stack, TLS buffers, and a frame buffer at the same time. The ESP32-S3-WROOM-1 has PSRAM built in so it can actually handle all of that. That decision alone probably saved me from a board that would OOM every time album art loaded.
Finding parts took longer than I expected. I wanted everything on JLCPCB's basic parts list to keep costs down. Spent a while searching for a basic USB-C connector with data lines, there was none. Ended up just putting the extended fee for three parts, the ESP32, the NT3H2111 NFC chip, and the USB-C connector. The NFC chip I picked is the NT3H2111. It has an I2C interface so the ESP32 can write to it directly. Every time Lanyard fires a track update the ESP32 rewrites the NDEF URL to the Spotify track link. When nothing is playing it writes my GitHub URL instead. I also wired the Field Detect pin to a GPIO so the firmware knows when someone is actively tapping the card.
Schematic went mostly fine. The power tree was straightforward, USB-C with 5.1k CC resistors, AMS1117-3.3 LDO, decoupling caps everywhere. The ESP32 programs over its native USB peripheral on IO19 and IO20 so no CH340 or CP2102 needed. That saved an extended part fee and simplified the board.
The antenna schematic took a little too long to get right. The NT3H2111 has two antenna pins, LA and LB. I kept going back and forth on whether they should be two different nets or the same net. The answer is they're the same net, the antenna coil is a continuous loop from LA around the board back to LB, and since KiCad won't route between different nets I had to make them both the same net. The 56pF tuning cap goes from ANT to GND. Took way longer than it should have to figure that out.
